Приложение по учету расходов.
Пользователь может создавать категории расходов, заносить свои траты. Показывается суммарное значение трат.
Данные кторые обрабатываются приложением
id (Целое число) - уникальный идентификатор записи (Primary Key).
amount (Вещественное число) - сумма расхода.
category (Строка) - категория трат (например: "Продукты", "Транспорт", "ЖКХ", "Развлечения").
date (Дата/Время) - дата совершения покупки.
description (Строка) - краткий комментарий или пояснение к трате (необязательное поле).
Действия в приложении
Создание (Create): Добавление новой записи о расходе через HTML-форму (ввод суммы, выбора категории и даты).
Чтение (Read): Просмотр таблицы/списка всех своих расходов. Отображение общей суммы всех трат.
Обновление (Update): Редактирование существующей записи (например, если пользователь ошибся при вводе суммы или категории).
Удаление (Delete): Удаление ошибочно созданной записи о расходе.

## Структура базы данных

База данных состоит из двух связанных таблиц: `users` (пользователи) и `expenses` (расходы). Связь реализована как «один-ко-многим» (у одного пользователя может быть много расходов).

### Таблица 1: `users` (Пользователи)
Описывает базового пользователя системы.
*   `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT) - Уникальный идентификатор пользователя.
*   `username` (TEXT, NOT NULL, UNIQUE) — Имя пользователя (логин).

### Таблица 2: `expenses` (Расходы)
Описывает основную сущность - запись о потраченных средствах.
*   `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT) - Уникальный идентификатор расхода.
*   `user_id` (INTEGER, NOT NULL) - Внешний ключ (FOREIGN KEY). Ссылается на `id` в таблице `users`. Указывает, чья это трата.
*   `amount` (REAL, NOT NULL) - Сумма расхода (число с плавающей точкой).
*   `category` (TEXT, NOT NULL) - Категория (например, "Еда", "Транспорт").
*   `date` (TEXT, NOT NULL) - Дата расхода в формате YYYY-MM-DD.
*   `description` (TEXT) - Необязательное текстовое описание/комментарий.
